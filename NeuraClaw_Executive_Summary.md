THE VERIFICATION PHILOSOPHY (Your New Quality Gates)
═══════════════════════════════════════════════════════════════════════════════════

Every feature must pass these 6 tests:

1. REALITY TEST: Does this render real computation, or is it fake?
   → If fake, it doesn't belong.

2. AGENCY TEST: Does this increase the pet's autonomy, or just add decoration?
   → If decoration, reconsider.

3. TRANSPARENCY TEST: Can the user understand why the pet did this?
   → If opaque, add explanation.

4. ETHICS TEST: Could this be used to manipulate, guilt, or exploit?
   → If yes, redesign or remove.

5. PERFORMANCE TEST: Does this run at 60fps on a 5-year-old laptop?
   → If no, optimize or degrade gracefully.

6. ACCESSIBILITY TEST: Can someone use this with only a keyboard? 
   With reduced motion? With a screen reader?
   → If no, fix before shipping.
